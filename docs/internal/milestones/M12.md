## M12-S1 — UDP audit + pre-DNAT deny-event fix

**Entry criteria.** Fresh — first session of M12 (M11 closed at 79/79).

**Spec reference.** Ad-hoc — no design spec. Anchored on todo #29 body and `sandboxd/sandboxd/tests/m10_s3_end_to_end.rs:610` (Phase 8 test 2 currently asserts the wrong post-DNAT tuple with an explanatory comment).

**Rationale.** UDP support is claimed end-to-end across nftables sets, the deny-logger, Envoy chains, and mitmproxy non-applicability — but the deny-logger's UDP path can't recover the pre-DNAT destination because `IP_ORIGDSTADDR` doesn't expose it for conntrack-DNAT'd UDP (TCP works only because `SO_ORIGINAL_DST` does the equivalent). The existing Phase 8 test in `m10_s3_end_to_end.rs` knowingly asserts the wrong tuple with a comment documenting the workaround — a quiet acknowledgement that UDP discovery is broken. M12 starts here because every subsequent UDP improvement (S2's e2e tests, S3's spec coherence) depends on having correct deny-event attribution first.

**In scope.**

Output: an audit notes file (`docs/internal/m12-s1-udp-audit.md`) capturing the current path map and any latent issues for downstream sessions.
- Fix #29: replace the IP_ORIGDSTADDR-only path in the UDP deny-logger handler with a conntrack netlink lookup for the original destination (likely via `nfnetlink_conntrack` or equivalent) so deny events carry the pre-DNAT 5-tuple.
- Rewrite the Phase 8 test 2 assertion in `sandboxd/sandboxd/tests/m10_s3_end_to_end.rs` to expect the pre-DNAT tuple. Remove the explanatory comment that documented the workaround.
- Add at least one new integration test asserting deny-event pre-DNAT attribution under load.

**Explicitly deferred.**

- Anything surfaced in audit but off the deny-attribution path → record in audit notes for S2 or S9.

**Exit criteria.**

- Full test suite passes (`make test` + `make test-integration`).
- No regression in TCP deny-logger behavior.
- Audit notes committed at `docs/internal/m12-s1-udp-audit.md`.

## M12-S2 — UDP datapath honesty + kernel-layer loggers

**Entry criteria.** M12-S1 (deny-event attribution must be correct before the datapath rework can move it from a userland conntrack lookup to a kernel-emitted NFLOG signal).

**Spec reference.** `.tasks/specs/2026-05-01-udp-nft-loggers/2026-05-01-udp-nft-loggers-design.md` (the M12-S2 design spec; referenced for full rationale, decisions, and architecture). Audit context lives at `docs/internal/m12-s1-udp-audit.md`.

**Rationale.** M12-S1 fixed UDP deny-event attribution by adding a conntrack-netlink lookup inside the deny-logger's UDP receive loop, but the audit it depended on surfaced three deeper truths: the UDP allow path through Envoy is structurally dead (Envoy has no UDP listener; the input chain rejects `udp dport 10000`), the UDP deny path runs a userland listener whose only job is to launder the kernel's own bookkeeping, and there is no audit-flow logging for allowed UDP at all. The original design at `networking-design.md:1395` ("UDP policy enforced purely by nftables ... no userland proxy") was never literally true. M12-S2 is a structural correction, not a feature addition: allowed UDP goes direct to upstream through nft (no Envoy hop), denied UDP becomes `nft drop` + NFLOG (no userland listener, no DNAT, no conntrack lookup), and a new kernel-conntrack subscriber closes the allow-flow audit gap. The deny-logger binary splits into a shared lib crate (`sandbox-event-emitter`) plus two `nft-`-prefixed binaries that signal the gating layer honestly. The original "test parity + doc reconciliation" framing is subsumed: the docs become reconcilable because the implementation finally matches what they always claimed.

**In scope.**

- **Allow-path datapath (Decision 1).** Reshape `sandbox_dnat` PREROUTING so a `policy_allow_udp` match `accepts` and falls through to MASQUERADE rather than DNATing to `gateway_ip:10000`. Allowed UDP exits the gateway directly to upstream; Envoy is not in the path. Touches `sandboxd/sandbox-core/src/gateway.rs:~1473` and `sandboxd/sandbox-core/src/policy.rs:~756`; forward-chain admit rule at `gateway.rs:~1565` extends to let allowed UDP through.
- **Deny-path datapath (Decision 2).** Replace the catch-all `dnat to gateway_ip:10002` rule with `nft log group N ; drop`. Delete the userland UDP listener (`sandbox-deny-logger/src/udp.rs::bind`, `::run`) and the `udp dport 10002 accept` input rule (`gateway.rs:1547`). The S1 conntrack-lookup module (`sandbox-deny-logger/src/conntrack.rs`) loses its only caller; deletion is the recommended outcome (Open Question #4). TCP-deny stays listener-based, byte-for-byte unchanged.
- **Allow-flow logger (Decision 3).** New userland subscriber on `nfnetlink_conntrack`'s `NFNLGRP_CONNTRACK_NEW` multicast group, filtered for UDP. Emits a JSONL allow event per new conntrack flow (per-flow, not per-packet). 30 s NFCT-rollover behaviour documented as expected, not asserted in tests.
- **Lib + binary rename (Decisions 4 + 5).** Extract `sandboxd/sandbox-event-emitter/` (lib only; owns `EventEmitter`, `RateCap`, flush ticker, `/health` shape, `DenyRecord`/`AllowRecord`, protocol enum). Rename `sandboxd/sandbox-deny-logger/` → `sandboxd/sandbox-nft-deny-logger/` (crate dir + workspace member + `[[bin]] name`). Add `sandboxd/sandbox-nft-allow-logger/` (binary + thin `main.rs` over the lib). Both ship in the gateway container; `networking/gateway/Dockerfile` and `networking/gateway/entrypoint.sh` extend to build, install, launch, monitor, and shutdown both.
- **Doc reconciliation (Decision 6).** Five docs updated: `docs/concepts/networking.md:141` (the `IP_ORIGDSTADDR` overclaim becomes a TCP-vs-UDP split phrasing), `networking-design.md:1395` (the "purely nftables" promise becomes literally true under the new datapath; the prose explaining the deny path updates to "observe-only NFLOG audit, not in the datapath"), `networking-design.md:1115-1121` (UDP stub replaced with substantive paragraph), `networking-design.md:1221` (ICMP-unreachable mandate reconciled to "silent drop" with rationale; gated on Open Question #2 — user must resolve before implementation), `docs/guides/network-policies.md` (new UDP section with concrete example), `docs/concepts/policy-model.md` (UDP-specific caveats next to the protocol enum), `docs/guides/troubleshooting.md` (UDP-specific entries: allow-event reading, 30 s NFCT rollover, silent-drop end-to-end shape).
- **Test parity (spec §Test plan).** New e2e cases under `tests/e2e/test_m4_policy.py` for allow-path delivery, bidirectional UDP echo, multi-port-same-host, allowed-IP edge case (direct-IP destination skipping DNS). New e2e case for deny attribution across the behavioural-blocking gap (`tests/e2e/test_m3_networking.py:440-444` skip-comment removed). Integration tests in `sandboxd/sandboxd/tests/`: allow-flow log assertion (allow event lands for an allowed flow; no event for a denied flow), deny-flow NFLOG log assertion (S1 wire-shape preserved; data source under test changes from "UDP listener + conntrack" to "NFLOG receive"). The S1 load test similarly retargets.

**Explicitly deferred.**

- ECHConfig handling fix → M12-S3.
- TCP datapath changes — out of M12-S2 scope. The TCP allow path through Envoy and the TCP deny listener with `SO_ORIGINAL_DST` + RST close move into `sandbox-nft-deny-logger` byte-for-byte; no shape change.
- Allow-logger `/health` audit-counter wiring (#112), conntrack recv-buffer hardening (#113), `iter_nlas` allocation reduction (#116) → M12-S9 candidates. The shared lib (Decision 5) makes (#112) tractable but doesn't perform it; (#113) and (#116) become moot if the conntrack module is deleted per Decision 2.
- Per-source UDP rate-cap on the new allow-logger — out of S2 scope. The per-process `RateCap` carries over from the deny-logger; per-source cap (re #108) defers to M12-S9 unless empirically needed.
- Anything UDP-related surfaced in the S1 audit but off the spec's six decisions → M12-S9.

## M12-S7 — Native `cp` via `limactl cp` / `docker cp`

**In scope.**
