# M10 — Port-explicit policies, presets, and observability

Harden the network policy model by making ports explicit per rule, replace the `Policy::unrestricted()` escape hatch with a CLI-local preset system, and expose a unified event surface across every policy-enforcing layer. Builds on the L3 flow landed in M9-S18..S20 and amends three specific decisions from that work (L3 destination_port predicate; VM-egress reject → `sandbox_dnat` conditional DNAT; Envoy access_log format). One release, one migration event (V004 SQLite migration; running sessions stop and restart at upgrade).

**Spec references:**

- `.tasks/specs/2026-04-21-port-explicit-policies-presets-observability-design.md` — primary spec; M10-S1..S6.
- `.tasks/specs/2026-04-19-l3-envoy-mitmproxy-flow-design.md` — receives amendment #1 in M10-S1 (destination_port on L3), #2 in M10-S3 (sandbox_policy VM-egress → sandbox_dnat), #3 in M10-S2 (access_log JSON format); spec-text edits for #1 and #2 land in M10-S6.
- [`2026-04-25-synchronous-dns-policy-gating-design/2026-04-25-synchronous-dns-policy-gating-design-spec.md`](../../../.tasks/specs/2026-04-25-synchronous-dns-policy-gating-design/2026-04-25-synchronous-dns-policy-gating-design-spec.md) — drafted in M10-S10; closes the DNS-rotation propagation race architecturally.

## M10-S1 — v2 port-explicit policy schema, compiler rewrite, and DB migration

Every downstream M10 session depends on v2 rules: S3's deny-logger uses the `policy_allow_{tcp,udp}` concat sets emitted here; S5's presets emit v2 rules; event shapes reference explicit port. **In scope.**

- Part 1 § "Compiler consequences → mitmproxy" — `PolicyCompiler::compile_mitmproxy` emits port; addon reads `flow.request.port`.
- Part 1 § "Compiler consequences → Path matcher semantics" — per-segment glob with `**` / `*` / `?`; replacement of `fnmatch.fnmatchcase` in the addon.
- Part 1 § "Schema bump and migration" — v1 parse hard-reject with prescribed error text; SQLite migration V004 (tighten `protocol` CHECK to `tcp/udp`; add `port` column with `CHECK (port BETWEEN 1 AND 65535)`; delete v1-shaped rows via table-copy). `policy_reset_on_upgrade` emitted as a tracing event here; re-wired to the ring buffer in S2.

- Known-gaps item: empirical check that `destination_port` on `FilterChainMatch` works with the pinned Envoy version's `original_dst` listener filter — first-step verification before building out the chain compiler, mirroring the M9-S19 tunneling-config pattern.

## M10-S3 — Deny-logger component and nftables DNAT restructure

**Entry criteria.** M10-S2 complete.

**Spec reference.** `.tasks/specs/2026-04-21-port-explicit-policies-presets-observability-design.md` — Part 3 §§ "Deny-logger component" (Placement in the nftables pipeline, Listener design, Hardening rules, Liveness posture); "Amendments to the L3 spec and its landed implementation" § 2.

**Rationale.** Closes the silent-denial gap acknowledged by the L3 spec. Today the VM-egress reject in `sandbox_policy` produces ICMP-admin-prohibited or TCP RST with no structured signal. This session replaces that reject with conditional DNAT in `sandbox_dnat` — allow → Envoy, TCP deny → deny-logger `:10001`, UDP deny → deny-logger `:10002` — using the `policy_allow_{tcp,udp}` concat sets from S1, and introduces a new gateway component whose sole job is structured logging of the deny. The `deny-logger` row in Part 3's traffic-event table is filled in here; events flow into the bind-mount JSONL already consumed by sandboxd from S2. Envoy-egress rules in `sandbox_policy` are preserved.

**In scope.**

- Part 3 § "Placement in the nftables pipeline" — cross-table restructure: `sandbox_dnat` absorbs VM-egress filtering via conditional DNAT rules keyed on the concat sets; `sandbox_policy` shrinks to Envoy-egress allow rules only; atomic `nft -f` multi-table transaction; `policy_distributor` extended to inject into `sandbox_dnat` alongside `sandbox_policy`; UDP set stays effectively empty until a UDP use case lands (future-proofed, not load-bearing).
- Part 3 § "Listener design" — all three listeners bind on `NetworkInfo.gateway_ip` (not `127.0.0.1`, which would require `route_localnet=1`); TCP listener `:10001` with `getsockopt(SO_ORIGINAL_DST)` + `SO_LINGER {1, 0}` RST close + never `recv`; UDP listener `:10002` with `IP_RECVORIGDSTADDR` cmsg + no response; healthcheck listener `:10003` with `GET /health` (not in the nftables DNAT set — container-internal).
- Part 3 § "Hardening rules" — invariants 1–6: no peer-controlled bytes in event fields, fixed-size buffers, TCP RST close, UDP body discarded, per-session rate cap, per-session TCP concurrent-connection cap. Cap breaches surface as periodic summary events.
- Part 3 § "Liveness posture" — Docker `HEALTHCHECK` probes `/health`; sandboxd's gateway health polling observes container-unhealthy and restarts; `health_degraded` / `health_restored` lifecycle events bracket the outage.
- L3 amendment #2 implementation (doc-text edit lands in M10-S6).
- `networking/gateway/` wiring — new component process, `entrypoint.sh` update, Dockerfile changes if any, `docker run` bind-mount from sandboxd.

**Explicitly deferred.**

- Supervised restart of only the deny-logger process (entry point `networking/gateway/entrypoint.sh:113-220`) → future, per spec "Out of scope".
- nftables packet logging in `sandbox_policy`'s Envoy-egress chain → out-of-scope per spec.
- L3 spec text edit for amendment #2 → M10-S6.
- Default numeric values for rate / connection caps → M10-S6.

**Exit criteria.**

- cargo nextest green; `make test-integration` green; `make test-e2e` full suite green.
- New integration test (TCP): VM opens a connection to a non-allow-listed destination → RST close; a `deny` event with `orig_dst_ip`, `orig_dst_port`, `protocol: tcp`, `src_ip`, `src_port` lands in the per-session ring buffer.
- New integration test (UDP): VM sends a datagram to a non-allow-listed destination → `IP_ORIGDSTADDR` cmsg surfaces as `orig_dst_ip`; event lands in ring buffer; no response emitted.
- Integration test: exactly three nftables tables present after session start (`sandbox`, `sandbox_dnat`, `sandbox_policy`); `sandbox_policy` VM-egress reject rules are gone.
- HEALTHCHECK green; `health_degraded` / `health_restored` events emit around an intentionally-injected listener failure.
- `/review` pass clean; `make docs-build` green.

## M10-S10 — Synchronous DNS-policy gating, workaround removal

**Entry criteria.** M10-S9 complete.

**Spec reference.** `.tasks/specs/2026-04-25-synchronous-dns-policy-gating-design/2026-04-25-synchronous-dns-policy-gating-design-spec.md` — drafted as the first deliverable of this session, then implemented in the same session.

**Rationale.** Today's DNS-driven policy propagation has a ~2-second blind window: the CoreDNS `sandboxpolicy` plugin resolves a new IP and answers the VM in under 1ms, but sandboxd's `dns_propagation_loop` polls `resolved.json` every 2 seconds, so the VM's TCP arrives at the gateway's nft set before the new IP is admitted and gets RST'd. M10-S8 worked around this for two known CDNs by pre-allowing entire Anycast supernets (`CARGO_FASTLY_CIDR_POOL` at commit 9d1dca7 and `GITHUB_INTERACTIVE_CIDR_POOL` at 874a9bb), but the workaround does not generalize to arbitrary user-policy domains and inflates the policy surface beyond what was authorized. This session closes the race architecturally by introducing synchronous gating between the CoreDNS plugin and sandboxd: the plugin holds the DNS response until sandboxd has acknowledged the nft update for the resolved IPs. Adds a daemon round-trip on first resolution of a new IP per host (~10–30 ms expected; user accepts), preserves sandboxd as the sole nft writer (no cross-language ownership split), and lets the existing E2E trio (`test_clone_repo`, `test_cargo_preset_allows_cargo_fetch`, `test_github_repo_preset_scopes_to_one_repo`) become the regression gate once the CIDR-pool workarounds are removed. The session co-produces the design spec and the implementation because the spec settles the IPC mechanism, ack semantics, and failure-mode contract that the implementation has to honour.

**In scope.**

- **Spec authoring.** Draft `.tasks/specs/2026-04-25-synchronous-dns-policy-gating-design/2026-04-25-synchronous-dns-policy-gating-design-spec.md`. Required sections: problem statement (race anatomy + measurements from M10-S8 progress log discoveries); goals and non-goals (workaround removal in scope; generalizing beyond the DNS path out of scope); IPC mechanism choice (FIFO pair, unix-domain socket in shared bind-mount, or HTTP on the gateway↔host bridge IP — pick one with explicit rationale, including ownership of the channel and reconnect semantics); ack protocol (request shape, success / negative-ack / timeout cases); failure modes (sandboxd slow, sandboxd unreachable, sandboxd dead — what does the plugin do, what does the VM see; fail-open vs fail-closed posture and the rationale for whichever direction is chosen); latency budget (target P50 / P99 on the gating round-trip; explicit cap above which the plugin falls back to a permissive answer + emits a structured warning event); CIDR-pool removal and the existing E2E trio as the regression gate; documentation pointers for the user-facing networking guide.
- **Implementation per spec.**
  - Go-side change in the CoreDNS sandboxpolicy plugin (`networking/coredns-plugin/`): in `WriteMsg` (current handler at `handler.go:133-156`), after writing `resolved.json`, signal sandboxd over the chosen IPC channel and block on ack (within the spec's latency cap) before returning the answer to the VM.
  - Rust-side change in the daemon (`sandboxd/sandbox-core/src/dns_propagation.rs` + adjacent): listen on the IPC channel; on receipt of a resolved-IP signal, perform the nft update synchronously, then ack. Replace or repurpose the 2-second polling loop only insofar as the gating contract requires — no broader refactor.
  - Gateway container surface (`networking/gateway/`): the chosen IPC channel exposed in the shared bind-mount (`/var/log/gateway/...` or a sibling path) with whatever ownership / mode bits the plugin and daemon need.

Delete the helper `http_rule_cidr` if it has no remaining callers.
